1. Start with root `README.md` for project-level overview and quick start.
2. Read `BACKEND_README.md` for API/service details.
3. Read `FRONTEND_README.md` for UI/client details.
4. Use `API_REFERENCE.md` while integrating or testing APIs.
5. Run through `DEPLOYMENT_CHECKLIST.md` before every production release.
6. Review `Visual_reference.md` for quick architecture and structure orientation.
