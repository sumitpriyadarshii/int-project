# DataVerse Documentation Index

This folder contains detailed project documentation.

## Documents

- `BACKEND_README.md` - Backend setup, scripts, routes, deployment, and troubleshooting.
- `FRONTEND_README.md` - Frontend setup, scripts, API integration, deployment, and troubleshooting.
- `Team_module.md` - Combined team module document with role-specific responsibilities.
- `Team_module.pdf` - Final two-module submission PDF with role-indication image.
- `API_REFERENCE.md` - API route groups, auth pattern, and endpoint summary.
- `DEPLOYMENT_CHECKLIST.md` - Production readiness and deployment validation checklist.
- `Visual_reference.md` - Architecture and folder flow snapshots for quick understanding.

## Suggested Reading Order

1. Start with root `README.md` for project-level overview and quick start.
2. Read `BACKEND_README.md` for API/service details.
3. Read `FRONTEND_README.md` for UI/client details.
4. Read `Team_module.md` for role-specific module responsibilities.
5. Open `Team_module.pdf` for the role-specific module submission in PDF format.
6. Use `API_REFERENCE.md` while integrating or testing APIs.
7. Run through `DEPLOYMENT_CHECKLIST.md` before every production release.
8. Review `Visual_reference.md` for quick architecture and structure orientation.
